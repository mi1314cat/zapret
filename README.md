# 🟩 **Zapret2 Panel v5.1 **

## 🚀 Zapret2 Panel v5.1

v5.1 是 Zapret2 Panel 的“生产稳定版”，专为复杂网络环境、双栈 IPv4/IPv6、iptables-nft 混合系统、运营商强 DPI 干扰等场景设计。本版本在 v5 的基础上进行了深度修复与强化，确保在长期运行、极端网络、透明网关、旁路由等环境中保持稳定可靠。

---

## 🔧 **本次更新重点（v5 → v5.1）**

### **1. 完整修复策略参数切割问题**
- 旧版本使用 `for token in $line` 会导致带空格/引号的策略被拆分  
- v5.1 改为 **整行作为一个参数**  
- 彻底避免 nfqws2 因策略格式导致的静默崩溃

### **2. 修正 nftables 规则语法**
- 移除冗余的 `ip protocol tcp tcp dport` 写法  
- 改为标准 `meta l4proto tcp tcp dport { ... }`  
- 兼容旧版/严格模式 nft

### **3. table/chain 存在性检查**
- 避免重复创建导致的报错  
- 支持多次 reload / restart  
- 更适合 systemd 长期运行

### **4. 多 IP 遍历增强**
- 遍历所有网卡的 global IPv4/IPv6  
- 自动过滤空元素  
- 自动加入安全白名单  
- 适配多网卡、多 IP、IPv6 SLAAC、PD 环境

### **5. 自动检测 iptables-legacy / iptables-nft**
- 避免 nft 与 iptables-nft 混合污染  
- 自动选择最安全的后端  
- 兼容 OpenVZ / LXC / 老内核

### **6. nfqws2 强杀逻辑优化**
- 增加冷却期  
- 避免队列占用残留  
- 防止“服务在跑但不生效”

### **7. IPv6 CIDR 清洗增强**
- 更严格的 IPv6 正则  
- 支持 IPv6/CIDR 节点  
- 防止畸形输入导致崩溃

### **8. 更详细的日志与错误捕获**
- `/var/log/nfqws2.log`  
- 启动参数完整打印  
- 方便调试与问题定位

## 🧭 面板菜单功能说明（小白版）

### 4）配置监听端口（IPv4/IPv6）

**干嘛用的？**  
决定 Zapret2 要“盯住”哪些端口的流量，然后对这些端口上的连接做 DPI 绕过处理。

**通俗理解：**  
- 你把 **Argo / TUIC / HY2 / 节点** 开在哪个端口，这里就填哪个端口  
- 比如：  
  - Argo：`443` 或 `8443`  
  - TUIC：`443` / `10443` / 自定义  
  - HY2：`443` / `8443` / 自定义  

**怎么填：**

- 支持多个端口，用英文逗号分隔：`443,8443,7844`  
- 有 IPv4 / IPv6 四组：  
  - `TCP4_PORTS`：IPv4 的 TCP 端口（最常用）  
  - `UDP4_PORTS`：IPv4 的 UDP 端口（TUIC/HY2 常用）  
  - `TCP6_PORTS`：IPv6 的 TCP 端口  
  - `UDP6_PORTS`：IPv6 的 UDP 端口  

**不会填就这样：**

- 只用 IPv4：  
  - `TCP4_PORTS` 填你的节点端口  
  - `UDP4_PORTS` 填你的 TUIC/HY2 端口  
  - 其他保持默认或留空  

---

### 5）配置包处理数量（前 N 个包）

**干嘛用的？**  
控制 Zapret2 对每个连接的“前多少个数据包”做伪装/拆包，用来绕过 DPI。

**通俗理解：**  
- DPI 一般只看连接刚开始的几个包  
- 我们只在前 N 个包上动手脚，后面就放行  
- 这样既能绕过检测，又不影响后续速度和稳定性  

**几个参数：**

- `TCP_PKT_OUT`：**出站 TCP** 前多少个包做处理（最关键）  
- `TCP_PKT_IN`：入站 TCP 前多少个包  
- `UDP_PKT_OUT`：出站 UDP 前多少个包  
- `UDP_PKT_IN`：入站 UDP 前多少个包  

**推荐：**

- 默认值已经是通用推荐：  
  - `TCP_PKT_OUT=9`  
  - `TCP_PKT_IN=3`  
- 一般不用改，除非你在极端网络下测试不同策略  

---

### 6）节点域名/IP 管理

**干嘛用的？**  
给 **某些节点单独做 DPI 绕过**，支持：

- 域名节点（如 Argo、VLESS、Trojan、WS、H2）  
- IP 节点（如 TUIC、HY2、Hysteria 纯 IP 节点）  

**你可以做什么：**

1. **添加节点白名单**  
   - 选择“域名节点”：  
     - 比如填：`argo.example.com`、`my-node.domain.com`  
     - 即使你乱填 `https://argo.example.com:443/path`，底层也会自动清洗成域名  
   - 选择“IP 节点”：  
     - 比如填：`1.2.3.4` 或 `2606:4700::`  
     - 适合 TUIC / HY2 / 纯 IP 节点  

2. **查看已有节点**  
   - 列出你已经添加过的节点名称（如 `argo`、`tuic`、`hy2` 等）

3. **删除节点**  
   - 删除某个节点对应的域名/IP 列表  

**通俗理解：**  
- 你可以给每个节点起一个名字，比如 `argo`、`tuic`、`hy2`  
- 每个名字下面可以挂一个域名或一个 IP  
- 所有这些节点会被自动合并到一个总列表里，统一做 DPI 绕过  

---

### 7）配置全局策略

**干嘛用的？**  
决定 Zapret2 用什么“手法”去骗过 DPI，比如：

- 伪造 TLS 握手  
- 拆分数据包  
- 打乱顺序  

**你有几种模式可以选：**

1. **Minimal（最小干扰）**  
   - 改动最少  
   - 适合网络环境比较宽松、只需要轻微绕过的情况  

2. **Stable（推荐）**  
   - 兼顾绕过效果和稳定性  
   - 大多数用户直接用这个就行  

3. **Aggressive（激进）**  
   - 用更多手段对抗强 DPI  
   - 适合“怎么连都连不上”的极端环境  
   - 可能对某些应用有兼容性影响  

4. **自定义输入**  
   - 给进阶用户用的  
   - 你可以自己写一整行 `--lua-desync=...` 参数  
   - 写错了可能导致服务启动失败  

**不会选就这样：**

- 直接选 **2）Stable（推荐）** 就行  

---

### 8）切换 Local / Gateway 模式

**干嘛用的？**  
决定 Zapret2 是只管“本机自己发出去的流量”，还是也管“经过本机转发的流量”。

**两种模式：**

- `local` 模式：  
  - 只处理本机发出的流量（OUTPUT）  
  - 适合：  
    - 你在 VPS 上跑 mihomo/sing-box  
    - 只让这台 VPS 自己走代理  

- `gateway` 模式：  
  - 处理经过本机的流量（PREROUTING + FORWARD）  
  - 适合：  
    - 旁路由  
    - 网关  
    - 透明代理  
    - 家里所有设备都通过这台机子上网  

**不会选就这样：**

- 如果你是 **VPS + mihomo + Argo**：用 `local`  
- 如果你是 **旁路由 / 网关**：用 `gateway`  

---

### 9）运行 Blockcheck

**干嘛用的？**  
用 Zapret 自带的检测脚本，测试你的网络到底是怎么被干扰的，比如：

- 是不是只重置 TLS  
- 是不是只拦截 SNI  
- 是不是只针对某些端口  
- 是不是只针对某些域名  

**它会做什么：**

1. 暂时清理 Zapret2 的防火墙规则  
2. 运行 `blockcheck2.sh` 或 `blockcheck.sh`  
3. 测试各种组合，输出检测结果  
4. 测试结束后自动恢复 Zapret2 的规则  

**什么时候用：**

- 你发现：  
  - 某些节点怎么都连不上  
  - 换策略也不稳定  
  - 不确定运营商到底在搞什么鬼  

就可以跑一次 Blockcheck，看检测结果，再针对性调策略。
